# 🐾 Fundamentos de Programación Orientada a Objetos (POO) 🐾

¡Bienvenido a este emocionante proyecto de aprendizaje sobre **Programación Orientada a Objetos** en Python! 🚀 Este repositorio demuestra los conceptos básicos de POO a través de una jerarquía de clases que representan seres vivos, mamíferos y personas. Perfecto para estudiantes y entusiastas que quieren dominar la herencia, encapsulación y polimorfismo.

![Python](https://img.shields.io/badge/Python-3.8%2B-blue?style=for-the-badge&logo=python)
![OOP](https://img.shields.io/badge/OOP-Herencia%20%26%20Polimorfismo-green?style=for-the-badge)
![License](https://img.shields.io/badge/License-MIT-yellow?style=for-the-badge)

## 🌟 Características Destacadas

- **Jerarquía de Clases**: Explora la herencia desde `SerVivo` hasta `Persona`
- **Métodos Interactivos**: Alimenta, duerme y trabaja con tus objetos
- **Encapsulación**: Atributos privados y métodos públicos
- **Polimorfismo**: Sobrescritura de métodos en subclases
- **Fácil de Entender**: Código comentado y estructurado

## 📁 Estructura del Proyecto

```
📦 fundamentos-poo/
├── 📄 main.py                 # Archivo principal con ejemplos de uso
├── 📂 seres_vivos/
│   ├── 📄 __init__.py         # Paquete para seres vivos
│   └── 📄 ser_vivo.py         # Clase base SerVivo
├── 📂 mamiferos/
│   ├── 📄 __init__.py         # Paquete para mamíferos
│   └── 📄 mamifero.py         # Clase Mamifero (hereda de SerVivo)
└── 📂 personas/
    ├── 📄 __init__.py         # Paquete para personas
    └── 📄 persona.py          # Clase Persona (hereda de Mamifero)
```

## 🛠️ Requisitos

- **Python 3.8+** 🐍
- No se requieren librerías externas (solo Python estándar)

## 🚀 Instalación y Uso

1. **Clona o descarga** este repositorio en tu máquina local.

2. **Ejecuta el programa** desde la raíz del proyecto:

   ```bash
   python main.py
   ```

3. **¡Disfruta!** Verás información sobre diferentes seres vivos en la consola.

## 🎯 Conceptos Aprendidos

- **Herencia**: `Mamifero` hereda de `SerVivo`, `Persona` de `Mamifero`
- **Encapsulación**: Atributos como `alive`, `warm_blooded`
- **Polimorfismo**: Métodos `eat()`, `sleep()` sobrescritos
- **Constructores**: Uso de `super()` para inicializar clases padre
- **Métodos de Instancia**: Comportamientos específicos de cada clase
